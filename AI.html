Introduction

  • Early identification of cognitive decline is crucial for effective treatments
  • Traditional screening methods like MMSE and MoCA face challenges in accessibility and scalability
  • Large Language Models (LLMs) can provide scalable, cost-effective screening tools
  • LLMs can analyze language patterns to detect subtle cognitive impairments
  • This study examines a novel agentic AI workflow for screening patient charts for cognitive concerns
Improving Early Detection

Methodology: Clinical Setting & Data

Clinical Setting

  • Clinical notes from Mass General Brigham's Research Patient Data Registry (RPDR)
  • Date range: January 1, 2016 - December 31, 2018
  • Note types: history, physical exams, clinic visits, discharge summaries, progress notes

Study Cohort

  • 3,338 clinical notes from 200 patients
  • Binary classification: patients with vs. without cognitive concerns
  • Split into prompt refinement and validation datasets
  • Analysis performed: February - August 2024

Methodology: Study Design

The study utilized two parallel workflows to detect cognitive concerns:

Agentic Workflow

An automated process using specialized agents making inferences through communications and tools

  • Multiple agents working in concert
  • Self-improving through iterations
  • Fully automated classification process

Expert-Driven Workflow

Created as a benchmark, combining LLM capabilities with clinical expertise

  • Relied on neurologist expertise
  • Manual review of false positives/negatives
  • Iterative prompt refinement

Classification Process

  1. LLM operated on individual notes with initial prompt (P0): "Is this note indicative of any cognitive concern, yes or no?"
  2. Patient-level classification determined by aggregating responses for all their notes:
    • If ANY responses indicated cognitive concerns → WITH cognitive concerns
    • If ALL responses indicated absence → WITHOUT cognitive concerns
    • If outputs were misformatted/inconclusive → UNCERTAIN (excluded from analysis)
  3. Performance metrics calculated against chart-reviewed (ground truth) labels

Agentic Workflow: Overview

The agentic workflow comprised six specialized agents working together to automatically refine prompts and enhance LLM capabilities:

Specialist

Expert in evaluating patients with cognitive concerns. Takes prompt and clinical notes as input, generating yes/no answers with reasoning.

Evaluator

Determines labels by aggregating responses across patient notes. Compares to reference labels and generates evaluation metrics.

Specificity Improver

Analyzes false positive examples to identify reasons for misclassification, refining prompts to increase specificity.

Sensitivity Improver

Focuses on false negative cases, seeking evidence of cognitive concerns that the LLM may have missed.

Summarizer 1

Summarizes improvements suggested by the Specificity Improver for each case into one new improved prompt.

Summarizer 2

Summarizes findings from the Sensitivity Improver and refines prompt by incorporating findings and SOP.

Iterative Process

  • If sensitivity < 0.8: Task directed to Sensitivity Improver → Summarizer 1 → Specialist
  • Similar process followed for specificity enhancement
  • Maximum of three iterations configured by default
  • Process concludes when:
    • Both sensitivity and specificity ≥ 0.8, OR
    • Change in sensitivity < 0.1

Agentic Workflow: Prompts & Iterations

The agentic workflow required only two iterations to achieve minimum thresholds for sensitivity and specificity (0.8).

Initial Prompt (P0)

System: You are a neurologist.

User: Is this note indicative of any cognitive concern, yes or no?

Agent Prompt 1 (AP1)

Is this note indicative of any cognitive concern, yes or no? Please consider the patient's medical history, symptoms, and physical examination findings, as well as any relevant keywords or phrases, to determine if there are any red flags for cognitive impairment or decline.

Agent Prompt 2 (AP2)

Is this note indicative of any cognitive concern, yes or no?

This question can be answered by reviewing the note for mentions of:

  • Medications commonly used to treat cognitive impairment
  • Diagnoses or symptoms related to dementia, Alzheimer's disease, or memory issues
  • Cognitive assessments (e.g. MOCA, MMSE)
  • Behavioral symptoms
  • Reported cognitive changes or objective evidence of impairment
  • Preserved independence in functional abilities
  • Absence of delirium or major psychiatric disorders

If any of these criteria are met, the answer is "yes". Otherwise, the answer is "no".

Iteration Process Findings

  • First iteration: SOP provided to Summarizer 1 instead of Specificity Improver
    • Resulted in unexpected outcomes - specificity decreased from 0.2 to 0.05
  • Second iteration: SOP provided to Specificity Improver
    • Successfully increased specificity from 0.20 to 1.00
    • AP2 classified 33 cases as uncertain
    • F1-score increased from 0.70 to 0.91
    • Sensitivity decreased from 0.94 to 0.84
    • Significant improvements in accuracy (0.59 → 0.90) and PPV (0.56 → 1.00)

Expert-Driven Workflow Presented by: Tanvir Ahmed Eamon

Workflow Process

  1. Begin with initial prompt (P0) for individual notes
  2. Aggregate responses to classify patients
  3. Generate evaluation metrics report
  4. Expert review of false positives & negatives
  5. Revise prompts based on clinical expertise
  6. Submit improved prompts to LLM for testing
  7. Further optimize using ChatGPT-4o

Expert-Driven Results

Required 4 iterations to achieve optimal performance

Final prompt (XP4) achieved F1-score of 0.90

Balanced sensitivity (0.87) and specificity (0.91)

High precision: PPV (0.93) and NPV (0.93)

Overall accuracy of 89%

Key Insight

Expert-driven workflow achieved high performance but required more iterations and clinical expertise compared to the agentic approach.
